underscore.js sortBy() 排序问题

Posted on 2015-08-13   |   In javascirpt   |  

最近在开发中有个需求，是要对数组中的对象，按照多个属性排序。
举个栗子：

var data = [
  {"level" : 1, "order" : 2},
     {"level" : 2, "order" : 2},
     {"level" : 1, "order" : 1},
     {"level" : 2, "order" : 1}
  ];


var output = [
  {"level" : 1, "order" : 1},
     {"level" : 1, "order" : 2},
     {"level" : 2, "order" : 1},
     {"level" : 2, "order" : 2}
  ];

就是array按照level, order进行ASC排序，输出output。
用SQL的话就是 order by level ASC, order ASC

第一个想到就是用underscore.js。

于是找到了underscore.js的sortBy, 看看官方文档描述

sortBy _.sortBy(list, iteratee, [context])
返回一个排序后的list拷贝副本。如果传递iteratee参数，iteratee将作为list中每个值的排序依据。迭代器也可以是字符串的属性的名称进行排序的(比如 length)。

所以我马上就用起来了

output = _.sortBy(data, 'level');
// 或者
output = _.sortBy(data, function(item){
    return item.level;
});

输出:

[
    {"level" : 1, "order" : 2},
    {"level" : 1, "order" : 1},
    {"level" : 2, "order" : 2},
    {"level" : 2, "order" : 1}
]

是按照level ASC排序了， 可是order并没有排序。

那就再改一下:

output = _.sortBy(array, ['level', 'order']);
// 或者
output = _.sortBy(array, function(item){
        return [item.level, item.order];
    });

输出:

      // 第一种方式
 [
    {"level" : 1, "order" : 2},
       {"level" : 2, "order" : 2},
       {"level" : 1, "order" : 1},
       {"level" : 2, "order" : 1}
    ]


      // 第二种方式
   [
   {"level" : 1, "order" : 1},
   {"level" : 1, "order" : 2},
   {"level" : 2, "order" : 1},
   {"level" : 2, "order" : 2}
]

第一种方式并没有排序，第二种正确的排序了。

于是我又去看了一遍underscore.js的文档和源码。发现：

_.sortBy() 接收的参数iteratee， 是用来作为排序的依据的。

排序的时候，会把iteratee应用到每个排序的对象中，当传递的是字符串，会用来当做对象的属性名称，排序时会取对应的值为依据排序，如果传递的是函数，则以返回的结果为依据排序。

所以第一种方式中 iteratee = ['level', 'order']参数是无效的。

第二种方式中，return [item.level, item.order]最后排序的时候会把数组转为字符串。所以当出现{'level' : 12, 'order : 12'}的数据时，这个数据会排在{'level' : 2, 'order' : 2}之前，因为排序的时候是’1212’, 跟’22’做比较，所以这种方式也不可以。

那到底要怎么做呢。再次看了一下underscore.js的sortBy()源码的实现，发现这里的sort是stable的。也就是说，当排序遇到相等的值，会保留原来的顺序。

举个栗子：

有这么些扑克牌 [ 方块5， 红桃9， 梅花5，黑桃4 ]
当我们以点数大小排序的时候，stable的排序，方块5的排序永远会在梅花5的前面，也就是保留之前的排序。unstable的排序怎会有可能出现梅花5在方块5前面的情况。

根据这个特性，我们在就可以这样子做 :

// 对order排序
data = _.sortBy(data, 'order');
// 对level排序
output = _sortBy(data, 'level');

输出结果: 

[
   {"level" : 1, "order" : 1},
   {"level" : 1, "order" : 2},
   {"level" : 2, "order" : 1},
   {"level" : 2, "order" : 2}
]

不过这样写太丑了，用underscore的链式语法连起来：

output = _.chain(data).sortBy('order').sortBy('level').value();

完美。

这里需要注意的点有:

  • 要先sortBy()次要属性 [order], 再sortBy()主要属性[level]。
  • 默认是ASC，做DESC排序的时候，要注意调用reverse()的位置。

再举个栗子:

要按照level DESC, order ASC 来排序，要这样子：

output = _.chain(data).sortBy('order').reverse().sortBy('level').reverse().value();

因为如果这样写：

output = _.chain(data).sortBy('order').sortBy('level').reverse().value();

先对order进行ASC排序， 再对level进行ASC排序，最后reverse()，这个时候会把order和level的排序都变成DESC。 所以要用第一种写法。
